Apktool

ApkTool官网

Apktool 于2010年3月1日发布第一版，已经连续更新 8 年，期间因Google提供的开源代码托管网站 Google Code 关闭，现已迁移至 Github。

Apktool 可以做什么？

  • 可以优雅地查看包内的资源文件
  • 修改smali文件（dex反编后的文件）
  • 重新打包，验证修改后的功能

简单方式

如果不想clone apktool 工程，可以在 Apktool Downloads 下载apktool jar包，推荐下载最新的，旧些的jar包反编市场上的apk包可能会失败。

有了jar包就可以像上面 java -jar apktool-xxxxxx.jar d myapp.apk反编apk包了，但这样每次输太繁琐了，别急，官方给了执行脚本。

右键下载 wrapper script

有了apktool jar 和 wripper script，将这两个文件分别重命名为apktool.jar和apktool，即：

apktool-xxxxxx.jar -> apktool.jar
下载的脚本 -> apktool

然后将两个文件移至 /usr/local/bin 目录（需要root权限），通过chmod +x分别给予执行权限。

到这时，就可以愉快的使用 apktool 命令执行反编和构建等操作啦。

Apktool 怎么用？

一个apk包，只不过是包含了资源和已组装后的java代码，通过 unzip 解压，可以看到AndroidManifest.xml、class.dex 和 resource.arsc等文件，包内的图片文件已经抽离放入assets文件夹中，但AndroidManifest和代码仍是编译后的，无法直接阅读，这时候就用到 Apktool 了。

打包后，需要签名，然后安装验证修改后的功能。

dex2jar 和 jd-gui

dex2jar github地址

配合使用dex2jar 和 jd-gui，可以方便我们反编和阅读代码。

dex2jar 可以做什么？

dex2jar 是用来处理 android .dex及 java .class文件的。

  1. dex-reader/writer: 用于读写 Dalvik 可执行文件 （.dex）。

  2. d2j-dex2jar: 将 .dex文件转换为 .class文件 （转换后的所有.class文件压缩为 jar）。

  3. smali/baksmali: 将 dex 反汇编到 smali 文件并从 smali 文件汇编 dex。 尽管与 smali/baksmali 语法相同，但实现方式不同，支持类型描述 Lcom/dex2jar\t\u1234;的转义。

dex2jar 如何获取？

代码 clone 及编译

编译成功后，在 dex-tools/build/distributions/ 下会生成zip文件，解压该文件，里面的 d2j-dex2jar.sh 文件就是我们要找的执行文件。

dex2jar 怎么用？

sh d2j-dex2jar.sh -f ~/path/to/your_apk_name.apk

默认会在当前目录生成 your_apk_name-dex2jar.jar文件，也可使用 -o <out-jar-file> 指定导出目录。

通过 dex2jar 工具将 apk 转成 jar后，这时候 JD-GUI 就可以出场了。
